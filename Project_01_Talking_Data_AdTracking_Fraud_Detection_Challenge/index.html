TalkingData AdTracking Fraud Detection Challenge¶

06 de maro, 2020

1. Descrição geral do problema¶

O risco de fraude est em toda parte, mas para as empresas que anunciam online, a fraude de cliques pode acontecer em um volume avassalador, resultando em dados de cliques enganosos e dinheiro desperdiado. Os canais de anncios podem aumentar os custos simplesmente quando pessoas ou bots clicam nos anncios em grande escala, o que na prtica no gera o resultado esperado. Com mais de 1 bilho de dispositivos mveis em uso todos os meses, a China o maior mercado mvel do mundo e, portanto, sofre com grandes volumes de trfego fraudulento.

A TalkingData, a maior plataforma de Big Data independente da China, cobre mais de 70% dos dispositivos mveis ativos em todo o pas. Eles lidam com 3 bilhes de cliques por dia, dos quais 90% so potencialmente fraudulentos. Sua abordagem atual para impedir fraudes de cliques para desenvolvedores de aplicativos medir a jornada do clique de um usurio em todo o portflio e sinalizar endereos IP que produzem muitos cliques, mas nunca acabam instalando aplicativos. Com essas informaes, eles criaram uma lista negra de IPs e uma lista negra de dispositivos.

Embora bem-sucedidos, eles querem estar sempre um passo frente dos fraudadores e desejam desenvolver ainda mais a soluo a partir da criao de um algoritmo que possa prever se um usurio far o download de um aplicativo depois de clicar em um anncio de aplicativo para dispositivos mveis.

Objetivo: Em resumo, neste projeto, iremos construir um modelo de aprendizado de mquina para determinar se um clique fraudulento ou no. Para a construo desse projeto, utilizaremos a linguagem R e o dataset disponvel no Kaggle em:

  • https://www.kaggle.com/c/talkingdata-adtracking-fraud-detection/data

2.2.2 Balanceando dados de treino¶

Precisamos balancear os dados de treino de maneira que as propores das classes da varivel is_attributed sejam aproximamente iguais. Isto , precisamos de um dataset que contenha a mesma quantidade de linhas com registros de indivduos que tenham ou no feito download.

Para solucionar este problema e ainda facilitar o manuseio destes dados na memria RAM, vamos criar um novo conjunto de dados de treino que contenha todos os registros do dataset original em que um download foi efetuado (is_attributed == 1). Com base no nmero de linhas obtidas na etapa anterior, iremos amostrar os registros do dataset original em que um download no foi efetuado (is_attributed == 0).

4. Feature Engennier¶

Precisamos preparar nosso dataset de treino para a fase de modelagem preditiva. Como a varivel attributed_time registra o instante de tempo em que o evento que desejamos prever ocorre, iremos remov-la do nosso conjunto de dados.

Como no vivel manipular diretamente as datas da varivel click_time, iremos apenas extrair os dias e as horas de cada registro e armazen-las em novas variveis (day e hour).

A varivel ip apresenta uma grande quantidade de valores nicos em relao as demais variveis e por isso, iremos apenas utiliz-la indiretamente para a criao de novas variveis a partir de agrupamentos.

In [28]:
# Criando e removendo variveis do dataset de treino.

data <- data %>%  
  select(-c(attributed_time)) %>%
  mutate(day = day(click_time), hour = hour(click_time)) %>% 
  select(-c(click_time)) %>%
  add_count(ip, day, hour)     %>% rename("ipDayHour"     = n)  %>%
  add_count(ip, hour, channel) %>% rename("ipHourChannel" = n) %>%
  add_count(ip, hour, os)      %>% rename("ipHourOs"      = n)  %>%
  add_count(ip, hour, app)     %>% rename("ipHourApp"     = n)  %>%
  add_count(ip, hour, device)  %>% rename("ipHourDevice"  = n)  %>%
  select(-c(ip))

5.2 Avaliando a importância das variáveis com o algoritmo Random Forest¶

Antes de iniciarmos a criao dos modelos propriamente dita, interessante definirmos o quanto cada varivel dentro do dataset ajuda a prever o valor da varivel target.

Para fazermos isto, utilizaremos o algoritmo Random Forest para computar estes valores. Como no sabemos qual o ajuste de parmetros do algoritmo mais adequado para o nosso conjunto de dados, criaremos modelos que possuam de 1 a 15 rvores com 1 a 40 ns.

Salvaremos a acurrica de cada modelo em um Dataframe. Note que por ser um processo de treinamento muito demorado, salvamos os resultados obtidos em um arquivo .csv que pode ser carregado com os resultados gerados pelo bloco de cdigo a seguir.

In [ ]:
# Criando o dataframe para salvar os resultados dos modelos.

featuresRF <- data.frame()

# Definindo o nmero de ns e rvores a serem combinados para a criao de diferentes modelos.

nTrees <- 1:15
nNodes <- 1:40

# Define o nmero total de modelos a serem criados.

total <- length(nTrees) * length(nNodes)

# Define uma varavel auxiliar para permitir o acompanhamento do progresso na avaliao dos modelos criados.

count <- 0

for(t in nTrees) {
    
  for(n in nNodes) {
    
    # Define um seed para permitir que os mesmos resultados dos experimentos sejam reproduzveis.
    
    set.seed(100)
    
    # Cria o modelo Random Forest a ser avaliado.

    model <- randomForest(is_attributed ~ .,
                          data       = data, 
                          ntree      = t, 
                          nodesize   = n, 
                          importance = T)
    
    # Computa a confusionMatrix gerada a partir do modelo criado.

    cm <- confusionMatrix(table(
        data      = model$y, 
        reference = model$predicted
    ))
    
    # Armazena os parmetros utilizados para criao do modelo e a acurcia obtida no dataframe.

    featuresRF <- rbind(featuresRF, data.frame(
        nodes    = n, 
        nTree    = t, 
        accuracy = unname(cm$overall['Accuracy'])
    ))
    
    # Incrementa o nmero de modelos avaliados.

    count <- count + 1

    # Imprime a porcetagem de progresso do treinamento e a melhor acurcia j alcanada.

    # print(paste(100 * count / total, '%, best accuracy: ', max(featuresRF$accuracy)))
  }

5.3 Criando modelos preditivos¶

Os modelos no paramtricos baseados em rvores de deciso tendem a apresentar um bom desempenho em um perodo de tempo de treinamento muito baixo quando o conjunto de dados muito grande. Por isso, iremos selecionar 4 algoritmos que sigam esta filosofia, so eles: Random Forest, C 5.0, Adaboost e XGboost. Tambm utilizaremos o algoritmo Naive Bayes por apresentar uma boa performance no treinamento de modelos com grandes quantidades de dados.

A estratgia que iremos utilizar para o treinamento dos modelos ser a de avaliar o desempenho de cada um utilizando os valores padres que possuem com poucas modificaes. Com base nos resultados que forem obtidos, iremos selecionar o melhor e otimizar seus parmetros.

Como o conjunto de dados de teste no possui a varivel target, utilizaremos o dataset de treino para fazer as predies e avaliar o desempenho do modelo, por isso, teremos que tomar mais cuidado com overfiting.
